# Qeet ID — Launch QA Findings

Master tracker for the July/August launch QA pass. See `qa/UX-AUDIT.md` for the parallel enterprise-UX consistency audit and the plan at large for phase sequencing.

## Legend

- **Type**: `Bug` (functional defect) · `Gap` (missing test coverage, not itself a defect) · `UX` (design-system inconsistency) · `Infra` (blocks testing itself) · `External` (needs 3rd-party/vendor action)
- **Severity**: `S1` launch-blocker · `S2` high (real screen/action broken or lies to the admin) · `S3` medium (workaround exists) · `S4` low (cosmetic/subjective)
- **Status**: `New` → `Confirmed` → `Fixed` → `Verified` → `Closed`, or `Deferred` / `Won't-Fix` (with reason)

## Automated route-audit tool

`go run ./tools/qa/route-audit` (from repo root) diffs every `api()`/`apiGet`/`apiPost`/`apiPatch`/`apiDelete` call-site in `apps/console/src` and `apps/login/src` against the merged OpenAPI route inventory (`api/openapi/*.yaml`) — the same technique that originally surfaced QID-01/QID-02, now repeatable. Run it first, before manual click-through, at the start of every module (per the per-module protocol below). `-md <path>` writes a markdown table instead of the human-readable stdout report; `qa/route-audit-report.md` is the last full-repo run's output (regenerate with `go run ./tools/qa/route-audit -md qa/route-audit-report.md`).

As of 2026-07-11, a global run over the whole console + login app surface found **zero unaccounted-for mismatches** — the 4 remaining (method, path) pairs it flags (`POST /v1/account/export`, `POST /v1/account/delete`, `POST /v1/admin/impersonate/exit`, `GET /v1/sso/discovery`) are all confirmed-graceful, self-documenting roadmap gaps (see QID-12) rather than bugs like QID-01/02 were.

**Phase 1 verdict:** core auth (password/passwordless/magic-links/passkeys config), passkey ceremony (browser + Go layers), TOTP enroll, and session revocation all verified working.
